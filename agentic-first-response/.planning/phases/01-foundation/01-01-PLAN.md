<task type="auto">
  <name>Task 3: Webhook endpoint, normalization, idempotency, health</name>
  <files>app/webhook.py, app/main.py</files>
  <action>
1. app/webhook.py:
   - parse_pagerduty(body: dict) -> NormalizedAlert | None: If payload looks like PagerDuty (e.g. has "routing_key" or "event_action" and "payload" or "dedup_key"), map to NormalizedAlert. Use dedup_key or payload.dedup_key or custom_details.incident_id as external_id; payload.summary or payload.custom_details for service/alert_type; payload.severity; source="pagerduty"; links from payload.links or custom_details.
   - parse_datadog(body: dict) -> NormalizedAlert | None: If payload looks like Datadog (e.g. "id", "title", "tags", "alert_type"), map to NormalizedAlert. id -> external_id; tags (service:*) -> service; title/alert_type; severity; source="datadog"; link -> links.
   - normalize(body: dict) -> NormalizedAlert: Try parse_pagerduty(body), then parse_datadog(body); raise ValueError if neither matches.
   - In-memory idempotency: module-level set (e.g. _seen_ids) and optional max size (e.g. 10000) or simple set; seen(external_id: str) -> bool, mark_seen(external_id: str).

2. app/main.py:
   - GET /health: return {"status": "ok", "service": "triage-agent"}.
   - POST /webhook/alert: Accept JSON body. Try normalize(body); if already seen(external_id) return same stub response (200) without re-calling agent. Else call run_triage(alert), mark_seen(alert.external_id), return StubTriageResponse as JSON. On ValueError (unknown payload) return 422 with {"detail": "Unsupported or invalid alert payload"}. Use Pydantic for response model so OpenAPI is correct.
  </action>
  <verify>
- GET http://127.0.0.1:8000/health → 200, body has status "ok".
- POST http://127.0.0.1:8000/webhook/alert with PagerDuty-shaped JSON (e.g. event_action, payload with summary, severity, dedup_key) → 200, body has hypothesis, suggested_action, etc.
- POST same body again (same dedup_key/external_id) → 200, same stub response (idempotent).
- POST with Datadog-shaped JSON (id, title, tags, alert_type) → 200, stub response.
- POST with invalid/malformed JSON or unknown shape → 422.
  </verify>
  <done>Health and webhook work; idempotency and at least one adapter (PagerDuty or Datadog) implemented; both adapters preferred per research.</done>
</task>

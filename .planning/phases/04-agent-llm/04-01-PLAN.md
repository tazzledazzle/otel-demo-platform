  <name>Task 1: Refine chain for tool-calling agent and add in-memory tools</name>
  <files>agent/agent/chain.py</files>
  <action>
- **Keep existing `search` tool:** Leave `@tool def search(query: str) -> str` in `chain.py` as-is (stub returning "[stub search result for: {query}]").
- **Add two in-memory-only tools (no DB/network):**
  - **Lookup tool:** e.g. `@tool def lookup(key: str) -> str` that consults a fixed in-memory dict (e.g. `DEMO_KNOWLEDGE = {"otel": "OpenTelemetry", "temporal": "Temporal workflow engine"}`). Return the value for key or a short "not found" message. No I/O.
  - **Echo or calc tool:** e.g. `@tool def echo_repeat(text: str, times: int = 1) -> str` that returns `(text + " ") * times` trimmed, or a simple deterministic utility. No I/O.
- **Introduce tool-calling agent:** Replace the current simple chain (`prompt | llm | _summarize_step`) with an agent that can call tools:
  - Use LangChain's agent factory that supports tool-calling: e.g. `create_react_agent` (from `langchain.agents`) or `create_tool_calling_agent` with a prompt, LLM with tools bound, and tools list `[search, lookup, echo_repeat]`. Prefer the API that matches the project's langchain/langchain-core versions (0.3.x): e.g. `AgentExecutor` with a tool-calling LLM and a prompt that instructs the model to use tools when helpful.
  - Ensure the agent's final output is a single string (e.g. last AIMessage content or agent executor result) so the existing `_Executor.invoke` contract `{"output": str}` can be preserved, or refactor to a small wrapper that invokes the agent and extracts the string reply.
- **Preserve get_agent_executor() contract:** `get_agent_executor()` must return an object with `.invoke(inputs: dict) -> dict` returning `{"output": "<reply string>"}` so `main.py` and tests do not break. If the new agent returns a different shape, wrap it in a thin adapter (e.g. same `_Executor` class or a new one) that maps agent result to `{"output": content}`.
- **Document the tool list and agent composition:** Add a short module docstring or comment in `chain.py` listing the tools and that the agent uses them when the model requests; reference CONFIG for OLLAMA_* and (later task) AGENT_LLM_OFF.
  </action>

Add or adjust one test that invokes the executor and asserts "output" in result. Optionally run agent manually with Ollama and send a message that should trigger a tool (e.g. "What is otel?") and confirm reply references lookup/search.</verify>

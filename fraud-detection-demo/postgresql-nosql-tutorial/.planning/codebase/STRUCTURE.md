## Directory Layout

```
postgresql-nosql-tutorial/
├── .planning/              # GSD planning artifacts
│   ├── PROJECT.md          # Project definition, value, constraints, decisions
│   ├── REQUIREMENTS.md     # Phase requirements, traceability (to be filled)
│   ├── ROADMAP.md          # Milestones, phases, progress table
│   ├── STATE.md            # Current phase/plan, status, progress
│   └── research/           # Optional research output

└── (no src/ or app code)   # Tutorial content not yet implemented
```

## Directory Purposes

**.planning/:**
- Purpose: GSD project metadata and planning; consumed by plan-phase, execute-phase, verifier, roadmapper.
- Contains: PROJECT.md, REQUIREMENTS.md, ROADMAP.md, STATE.md, `research/README.md`.
- Key files: All of the above; STATE.md is the primary session/continuity artifact.

**postgresql-nosql-tutorial/ (root):**
- Purpose: Project root; holds config and (future) source/tutorial content.
- Contains: `config.json` only at present; no `src/`, `labs/`, or `tests/` yet.

- Planning docs: UPPERCASE.md (PROJECT.md, REQUIREMENTS.md, ROADMAP.md, STATE.md).

- Research: `research/README.md` as placeholder.

**Directories:**
- `.planning/`: GSD convention for planning artifacts; `research/` under it for optional research.

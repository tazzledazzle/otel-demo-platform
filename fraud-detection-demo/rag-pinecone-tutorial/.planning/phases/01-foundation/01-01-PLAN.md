<task type="auto">
  <name>Task 2: Ingest pipeline (chunk, embed, upsert, verify)</name>
  <files>src/ingest.py</files>
  <action>
Implement the single-run pipeline in src/ingest.py. Use RESEARCH.md patterns and avoid dimension mismatch, missing index readiness, and unbatched upsert.

1. **Env and keys** — At top: load_dotenv(). Before any API call, ensure PINECONE_API_KEY and OPENAI_API_KEY are set (sys.exit with clear message if missing).

2. **Ingest** — Read all .md (and optionally .txt) files from docs/ (use pathlib.Path("docs").glob("*.md") or equivalent). Concatenate or process per file; keep track of source file for chunk IDs.

3. **Chunk** — Use RecursiveCharacterTextSplitter from langchain_text_splitters with chunk_size=500, chunk_overlap=50. Split each document's text; assign stable IDs per chunk (e.g. filename_stem + "_" + str(i)). Produce list of (id, text) or similar.

4. **Embed** — Call OpenAI embeddings API: model="text-embedding-3-small" (dimension 1536). Batch the chunk texts (OpenAI accepts list of strings); extract embedding vectors. Use a single constant EMBEDDING_DIM = 1536 for index creation and validation.

5. **Index** — Pinecone(api_key=os.getenv("PINECONE_API_KEY")). Index name e.g. "rag-tutorial". If not pc.has_index(name): create serverless index with dimension=1536, metric=Metric.COSINE, spec=ServerlessSpec(cloud=CloudProvider.AWS, region=AwsRegion.US_WEST_2). Wait for index ready (e.g. poll describe_index until ready or use short sleep; RESEARCH warns index creation is async). Get host from describe_index and get Index(host).

6. **Upsert** — Build vectors list: {"id": id, "values": embedding, "metadata": {"text": chunk_text}} for each chunk. Upsert in batches of up to 1000 vectors per request; use a namespace (e.g. "tutorial").

7. **Verify** — Call index.describe_index_stats() and print total_vector_count (and optionally namespace counts). Script should exit successfully after this.

Do not use Pinecone integrated embedding (create_index_for_model); keep bring-your-own vectors so the embed step is explicit per RESEARCH and CONTEXT.
  </action>
  <verify>Run from repo root: python src/ingest.py (or python rag-pinecone-tutorial/src/ingest.py). Script completes without error and prints total_vector_count >= number of chunks from sample docs. No dimension mismatch or "index not found" errors.</verify>

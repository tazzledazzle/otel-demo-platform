<task type="auto">
  <name>Task 2: Minimal FastAPI app with two operations</name>
  <files>main.py</files>
  <action>
Create main.py at project root. Use FastAPI (from fastapi import FastAPI). Define two GET operations:

1. GET / — root. Return JSON, e.g. {"message": "Hello World"} (or equivalent). Use a single path operation (e.g. @app.get("/")).

2. GET /greeting — second operation. Accept optional query parameter (e.g. name, default "World"). Return JSON, e.g. {"greeting": "Hello, {name}!"}. Use @app.get("/greeting") with a query param so the learner sees a second distinct endpoint.

Use async def for handlers. Return plain dicts; FastAPI serializes to JSON. Do not add auth, versioning, or OpenAPI customization. Per RESEARCH: use the framework's run command (fastapi dev main.py), do not hand-roll HTTP server.
  </action>
  <verify>Run `fastapi dev main.py` from project root (after pip install -r requirements.txt). Then: curl -s http://127.0.0.1:8000 returns JSON with a message; curl -s "http://127.0.0.1:8000/greeting" returns JSON with greeting; curl -s "http://127.0.0.1:8000/greeting?name=Alice" returns JSON with greeting for Alice. Server starts without error.</verify>
  <done>Two GET endpoints respond with JSON; single-command run works; /docs serves interactive API docs.</done>

## Overview

Tutorial project teaching software engineers Python and machine learning: data handling, model training, evaluation, and deployment. Phases TBD after research and requirement derivation.

## Phases

### Phase 1: Foundation — environment and first model
**Goal:** Learners get a runnable Python/ML environment and complete a first model (e.g. train/eval on a small dataset). No deployment yet.
**Success Criteria:** One lesson or notebook that runs end-to-end; clear setup steps; engineer-oriented (code-first, minimal theory).

**Plans:** 1 plan

Plans:
- [x] 01-01-PLAN.md — Environment and first lesson (executed)

### Phase 2: Evaluation and metrics
**Goal:** Learners go beyond single-number accuracy: add evaluation metrics (precision, recall, F1, confusion matrix) as text-only output. Still local, script; no deployment, no plotting.
**Success Criteria:** At least one runnable lesson (lesson2.py) that reuses Phase 1 (Iris, same split/model), computes classification_report and confusion_matrix, and prints to stdout; engineer-oriented.
**Depends on:** Phase 1

Plans:
- [ ] 02-01-PLAN.md — lesson2.py: Phase 1 reuse + classification_report and confusion_matrix (text-only)

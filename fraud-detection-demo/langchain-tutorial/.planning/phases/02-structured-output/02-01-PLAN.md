Implement one LCEL chain that returns a Pydantic instance instead of plain text. Do not use with_structured_output (ChatOllama does not support it).

1. **Pydantic model** — Define one small schema with 2–3 fields (e.g. title: str, summary: str, tags: list[str] or Optional[str]). Use Field(description=...) so format_instructions are useful. Place in same file or top of script.

2. **Parser** — Use PydanticOutputParser(pydantic_object=YourModel) from langchain_core.output_parsers. JsonOutputParser(pydantic_object=YourModel) is acceptable and yields same typed result.

3. **Prompt** — Use ChatPromptTemplate (consistent with 01_hello_chain.py). Include {format_instructions} in one message (e.g. human). Set format_instructions via prompt.partial(format_instructions=parser.get_format_instructions()) so the LLM is instructed to output valid JSON. One user-facing variable (e.g. {input}) for the text to summarize or the query.

4. **LLM** — Reuse Phase 1: ChatOllama(model="llama3.1", temperature=0) from langchain_ollama. No new providers.

5. **Chain** — Compose: prompt | llm | parser. Invoke with one input (e.g. "Summarize the following in one sentence and suggest a title and tags: ..." or a short user query). Print or use the result (e.g. print(result) or print(result.title, result.summary)).

6. **Dependencies** — Do not add new packages; pydantic is transitive from langchain. Add pydantic to requirements.txt only if missing in the project.

7. **Scope** — No agents, tools, or RAG. Single sync invoke().
